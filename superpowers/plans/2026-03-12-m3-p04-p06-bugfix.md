> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Tech Stack:** Vanilla JS ES Modules, GSAP (from ScrollAnimations.js), SortableJS

> **范围说明：** 本计划聚焦 S1（5步渐进美化）的粘性滚动区域。S2（案例对比）、S3（代码对照）、Hero、Footer 的动效不在本次范围内，已有 fadeIn ScrollTrigger 动画保持不变。

### Task 1: 修复 p04 粘性滚动布局抖动

**Files:**
- Modify: `src/pages/m3/p04-chart-beautify.js`（`cacheLayout`、`onStickyScroll` 函数，约第 996–1053 行）

**背景：** 当前 `onStickyScroll()` 每帧调用 `bodyEl.getBoundingClientRect()` 强制浏览器重新布局（layout reflow），紧接着写 `style.transform`，造成持续抖动。此外步骤用 `Math.floor(scrolledPast / innerHeight)` 计算，在边界附近来回震荡触发 `updateStepUI` 反复调用。

#### Bug 位置（约第 996–1053 行）

**Bug 1 — getBoundingClientRect 在热路径**：
```js
// ❌ 当前：每帧都读取 getBoundingClientRect() → 强制 layout reflow
const scrolledPast = Math.max(0, -bodyEl.getBoundingClientRect().top);
```

**Bug 2 — 边界震荡**：
```js
// ❌ Math.floor 在边界会震荡
const stepIdx = Math.min(..., Math.floor(scrolledPast / window.innerHeight));
```

- [ ] **Step 1: 在模块状态区添加 `_cachedBodyTop` 变量**

修改为：
```js

  const leftEl = document.getElementById('p04-steps-left');
```

- [ ] **Step 3: 重写 `onStickyScroll()` 消除热路径 DOM 读写（Task 1 阶段仍调用 `updateStepUI` 直接更新）**

将 `onStickyScroll()` 函数完整替换：
```js
function onStickyScroll() {
  const isMobile = window.innerWidth <= 768;
  if (isMobile) return;

  const leftEl = document.getElementById('p04-steps-left');
  if (!leftEl) return;

  const scrollY      = window.scrollY;
  const scrolledPast = Math.max(0, scrollY - _cachedBodyTop);
  leftEl.style.transform = `translateY(${Math.min(scrolledPast, _maxTranslate)}px)`;

  // 用视口中心判断步骤，防止边界震荡
  const midY    = scrollY + window.innerHeight * 0.5;
  const rawIdx  = Math.floor((midY - _cachedBodyTop) / window.innerHeight);
  const stepIdx = Math.min(BEAUTY_STEPS.length - 1, Math.max(0, rawIdx));

  if (stepIdx !== currentStep) {
    currentStep = stepIdx;
    updateStepUI(currentStep);   // Task 2 会替换为 animateStepTransition
  }
}
```

**注意：** Task 1 的 `onStickyScroll` 仍调用 `updateStepUI`（无动画），确保 Task 1 commit 后页面完全可用。Task 2 再将这行替换为 `animateStepTransition`。

- [ ] **Step 1: 在模块状态区添加 `_stepAnimating`，在 `updateStepUI` 后插入 `animateStepTransition`**

**先**在文件顶部的模块状态区（找到 `let currentStep = 0;` 附近）添加：

```js
function animateStepTransition(idx) {
  const numEl     = document.getElementById('p04-step-num');
  const titleEl   = document.getElementById('p04-step-title');
  const descEl    = document.getElementById('p04-step-desc');
  const codeEl    = document.getElementById('p04-step-code');
  const changesEl = document.getElementById('p04-changes');
  const targets   = [numEl, titleEl, changesEl, descEl, codeEl].filter(Boolean);

  if (_stepAnimating) {
    // 快速滚动时直接更新，跳过动画
    updateStepUI(idx);
    return;
  }
  _stepAnimating = true;

  gsap.to(targets, {
    opacity: 0, y: -10, duration: 0.18, ease: 'power2.in',
    onComplete() {
      updateStepUI(idx);
      gsap.fromTo(targets,
        { opacity: 0, y: 12 },
        {
          opacity: 1, y: 0, duration: 0.32, stagger: 0.05,
          ease: 'power3.out',
          onComplete() { _stepAnimating = false; },
        }
      );
    },
  });
}
```

**选方案 B：** 新增模块级变量 `let _poolSortable = null;`，pool Sortable 单独存放，不随 `_sortableInstances` 清理。

## 开发注意事项

### p04 关键规则
1. **`_cachedBodyTop` 必须在 `cacheLayout()` 内计算**，不允许在 `onStickyScroll()` 内读取 `getBoundingClientRect()`
2. **`animateStepTransition` 必须在 Task 1 的 `onStickyScroll` 引用它之前定义**（或确保两个 Task 在同一 commit）
3. **`_stepAnimating` 标志必须在 `destroy()` 中重置为 `false`**，否则再次进入页面动画永远跳过

### p06 关键规则
4. **`_poolSortable` 不进入 `_sortableInstances`**，这是核心设计原则——cell Sortables 每次重建，pool Sortable 整个页面生命周期只创建一次
5. **`destroy()` 必须同时清理 `_poolSortable`**，否则导航返回时 pool 已有旧实例
6. **`pool._hasSortable` 扩展属性完全废弃**，从所有代码中删除，改用 `_poolSortable !== null` 判断

### 测试顺序
1. 先测 p04 桌面端滚动（无抖动 + 动画）
2. 再测 p04 移动端（iOS 样式正确）
3. 测 p06 拖放（首次 + 切换布局后 + 导航返回后）
