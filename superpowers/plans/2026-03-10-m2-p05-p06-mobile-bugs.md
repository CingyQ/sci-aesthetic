# M2 P05/P06 移动端 Bug 修复计划

> **For agentic workers:** REQUIRED: Use superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复 p05 和 p06 页面在移动端的三类 Bug：容器溢出屏幕、表格超出、案例卡片不可见

**Architecture:** 问题根源有三：(1) fadeIn 动画导致案例卡片停留在 opacity:0；(2) 多个 section 缺少 overflow-x 保护；(3) 内联 style padding 覆盖了 layout.css 的移动端规则。逐一修复并在 Chrome DevTools 移动端模拟器中验证。

**Tech Stack:** CSS, GSAP ScrollTrigger, Chrome DevTools Mobile Emulation

---

## Bug 分析

### Bug 1: p06 S1 案例选择卡片不可见（"黑的"，但点击有反应）

**根因:** `init()` 中对 `.p06-case-card` 调用 `fadeIn()`，内部 `gsap.from(el, { opacity: 0, ... })` 立即将卡片设为 opacity:0。ScrollTrigger 在移动端可能因时序问题未触发动画，卡片永远停在 opacity:0。卡片 DOM 存在所以点击有响应，但视觉上不可见（整块区域显示深色背景）。

**文件:** `src/pages/m2/p06-case-studies.js:542-544`

### Bug 2: p06 案例二所有容器移动端溢出

**根因:**
- 所有 section 使用 **内联 style** `style="padding:var(--space-3xl) var(--space-xl);"` 覆盖了 layout.css 的移动端 padding 规则（`padding: var(--space-xl) var(--space-sm)`）
- 仅 `#p06-s3` 和 `#p06-s5` 有 `overflow-x:hidden`，`#p06-s1`、`#p06-s2`、`#p06-s4` 没有溢出保护
- Mermaid SVG 渲染后可能超出容器宽度

**文件:** `src/pages/m2/p06-case-studies.js:288, 342, 356, 388, 421, 445`

### Bug 3: p05 表格 + p06 汇总表移动端溢出

**根因:**
- p05: `#p05-s1` 有 `overflow-x:hidden`，`.p05-table-wrap` 有 `overflow-x:auto`，但缺少 `max-width:100%`
- p06: `.p06-summary-wrap` 在移动端有 `max-width:100%`，但 section `#p06-s5` 有 `overflow-x:hidden` 应该够用
- 核心问题：内联 padding style 使可用内容宽度变小，加上缺少 `max-width:100%` 导致表格容器超出

**文件:** `src/pages/m2/p05-ai-diagrams.js:179,189` 和 `src/pages/m2/p06-case-studies.js:288,314`

## Chunk 1: 修复所有 Bug

### Task 1: 修复 p06 案例卡片不可见

**Files:**
- Modify: `src/pages/m2/p06-case-studies.js` (init 函数中 fadeIn 调用 + CSS)

- [ ] **Step 1: 移除案例卡片的 fadeIn 动画，改用 CSS 动画**

在 `init()` 中，将 `.p06-case-card` 的 `fadeIn()` 调用替换为简单的 `gsap.fromTo`（不依赖 ScrollTrigger），确保卡片始终可见：

```js
// 原代码 (lines 542-544):
document.querySelectorAll('.p06-case-card').forEach((el, i) =>
  fadeIn(el, { delay: i * 0.1, y: 24, duration: 0.7 })
);

// 替换为直接 GSAP 动画（不依赖 ScrollTrigger，确保在移动端始终执行）：
document.querySelectorAll('.p06-case-card').forEach((el, i) =>
  gsap.fromTo(el,
    { opacity: 0, y: 24 },
    { opacity: 1, y: 0, duration: 0.7, delay: 0.3 + i * 0.1, ease: 'power3.out' }
  )
);
```

这样卡片动画不再依赖 ScrollTrigger 的视口检测，在页面 init 时直接播放。

### Task 2: 修复 p06 所有 section 移动端溢出

**Files:**
- Modify: `src/pages/m2/p06-case-studies.js` (HTML 和 CSS)

- [ ] **Step 2a: 移除所有 section 的内联 padding style**

将所有 section 的内联 `style="padding:var(--space-3xl) var(--space-xl);"` 移除，改为 CSS 类控制。这样 layout.css 的移动端 padding 规则可以正常生效。

影响的 section（5 处）：
- `#p06-s1` (line 342): 移除 `style="padding:var(--space-3xl) var(--space-xl);"`
- `#p06-s2` (line 356): 同上
- `#p06-s3` (line 388): 同上
- `#p06-s4` (line 421): 同上
- `#p06-s5` (line 445): 同上

在 `<style>` 块中添加桌面端 padding 规则（保持原有的桌面端视觉效果）：

```css
/* 添加到 style 块中（responsive 前面） */
#p06-s1, #p06-s2, #p06-s3, #p06-s4, #p06-s5 {
  padding: var(--space-3xl) var(--space-xl);
}
```

然后在移动端 `@media (max-width:768px)` 中覆盖：

```css
/* 确保移动端 padding 生效（已有 scroll-margin-top，在同一块内添加） */
#p06-s1, #p06-s2, #p06-s3, #p06-s4, #p06-s5 {
  scroll-margin-top: 56px;
  padding: var(--space-xl) var(--space-sm);
}
```

- [ ] **Step 2b: 扩展 overflow-x:hidden 到所有 section**

将 `overflow-x:hidden` 从仅 `#p06-s3, #p06-s5` 扩展到所有 section：

```css
/* 原代码 (line 288): */
#p06-s3, #p06-s5 { overflow-x:hidden; }

/* 替换为: */
#p06-s1, #p06-s2, #p06-s3, #p06-s4, #p06-s5 { overflow-x:hidden; }
```

- [ ] **Step 2c: 为 Mermaid 容器添加移动端 max-width 约束**

在 `@media (max-width:768px)` 中添加：

```css
.p06-mermaid-wrap { max-width: 100%; box-sizing: border-box; }
```

- [ ] **Step 2d: 为 before/after 行和图片容器添加移动端约束**

在 `@media (max-width:768px)` 中添加：

```css
.p06-ba-row { max-width: 100%; box-sizing: border-box; }
.p06-steps-grid { max-width: 100%; box-sizing: border-box; }
```
