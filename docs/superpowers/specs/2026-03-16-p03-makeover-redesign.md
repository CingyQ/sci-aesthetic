# P03 PPT 改造工坊 — 幻灯片重做设计

## 设计目标

重做 P03 的 6 组 Before/After 幻灯片内容，解决以下问题：
1. 当前内容太少太假，不像真实案例
2. After 内容过度精简（只剩 3 个数字），不真实
3. 6 个案例都绑定同一学科，缺乏普适性

## 核心原则

**Before 和 After 的信息量应该接近，差别在于组织方式和视觉层次。**

- Before：信息饱满但组织混乱 → 让人看一眼就觉得"这就是我做的那种 PPT"
- After：同等信息量但结构化呈现 → 视觉层次清晰、重点突出
- 不绑定具体学科，按 PPT 页面类型分类

## 6 个案例分类（完整 PPT 流程）

封面 → 目录 → 文字 → 数据表格 → 图表 → 结论

After 风格：封面/结论深色（#1d1d1f），中间 4 页白底。

## 技术约束

- 容器：`.mko-slide`，`aspect-ratio: 16/9`，`padding: 28px 32px`（移动端 `16px 18px`），`font-size: 13px`
- 所有内容用 inline style（因为在 JS 模板字符串中）
- Before/After 通过 `clip-path: inset()` 裁切（内容不变形）
- 保持 `width:100%; height:100%; box-sizing:border-box` 在最外层 div

## Case 1：封面页

### Before（学校模板丑）

视觉特征：
- 蓝色渐变背景 `linear-gradient(135deg, #c8d8ee, #dde8ff)`
- 顶部左右各一个元素（[校徽] + 大学名）
- 中间宋体加粗长标题（完整论文标题 25+ 字）
- 英文副标题
- 答辩人/导师/学院/课题组完整信息
- 底部蓝色色带 + 日期/次数/保密声明
- 两侧蓝色竖线装饰

内容：
```
[校徽]                                          某某大学

基于深度学习的多模态医学影像分析
与肿瘤区域自动分割系统关键技术研究进展汇报

Research Progress on Key Technologies of
Automatic Tumor Segmentation

汇报人：张三  |  指导老师：李教授
课题组：医学AI实验室 · 学院：计算机科学与技术学院

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
2024年12月 | 第xx次组会汇报 | 内部资料·请勿外传
```

### After（深色极简）

视觉特征：
- 深灰 `#1d1d1f` 背景
- 左侧暖橙竖线 4px
- 上方小标签 `letter-spacing: 0.12em`，灰色大写
- 大标题白色粗体，只保留核心研究对象（2 行）
- 下方姓名 + 导师（小字）
- 底部细线分隔 + 学校 + 日期

内容（信息量保持，层次重组）：
```
组会汇报                              （小标签）

多模态医学影像                        （大标题）
肿瘤区域分割

张三 · 导师：李教授                    （小字）
2024 · 12 · 15

────────────────────────────
医学AI实验室 · 计算机科学与技术学院      2024 · 12
```

### Takeaway
1. 标题砍到核心关键词——30 字标题提炼成 8 字，修饰词全删
2. 信息分三层：标题（最大）> 姓名导师（中）> 学校日期（最小），字号差 2 倍
3. 去掉渐变/水印/装饰线/色带——白底或深色底二选一

## Case 2：目录页

### Before（学校模板丑）

视觉特征：
- 白色背景
- 标题 "目 录" 居中，蓝色粗体带下划线
- 蓝色小圆点 + 编号列表
- 所有章节标题同一字号、同一颜色，没有层次
- 底部页码

内容：
```
               目  录
              ————————

  ● 一、研究背景与文献综述....................3
  ● 二、研究问题与假设.......................8
  ● 三、基于卷积神经网络的图像分割方法........12
  ● 四、实验设计与数据采集....................18
  ● 五、实验结果与分析.......................22
  ● 六、结论与展望...........................28

                                        2 / 30
```

### After（白底现代导航）

视觉特征：
- 白色背景
- 左侧超大编号数字（20px+），暖橙色
- 右侧章节标题，正常字号
- 当前章节（第三章）用暖橙色高亮 + 略大字号
- 其余章节用浅灰色
- 无圆点、无页码、无下划线
- 充足行间距

内容（同样 6 章，但有视觉重点）：
```
01  研究背景与文献综述                 （灰色）
02  研究问题与假设                     （灰色）
03  基于 CNN 的图像分割方法             （暖橙高亮，大字）
04  实验设计与数据采集                 （灰色）
05  实验结果与分析                     （灰色）
06  结论与展望                         （灰色）
```

### Takeaway
1. 目录是导航不是清单——高亮当前章节，让观众知道"我们在哪"
2. 用大数字替代圆点，建立视觉锚点，观众扫一眼就能定位
3. 删掉页码和省略号——观众不需要翻页找内容

## Case 3：文字页

### Before（手残丑）

视觉特征：
- 浅灰背景 `#f5f5f5`
- 标题红色加粗 + 下划线
- 正文三种颜色混用（黑/蓝/深红）
- 段落里随机加粗、随机斜体
- 行距 1.0，段间距为零
- 两段密排文字，约 200 字

内容：
```
研究方法（红色加粗下划线）
————————————————————
本研究采用随机对照双盲实验设计，招募符合纳入标准
的受试者共72名，随机分配至实验组（n=36, 蓝色）和
对照组（n=36, 蓝色）。实验组接受为期8周的干预方案，
每周两次（红色加粗），每次45分钟。干预内容包括
认知行为训练、生物反馈技术以及个体化方案调整。
对照组接受同等时间的安慰剂控制处理。
主要结局指标（斜体）为认知功能综合评分（MMSE、
MoCA），次要指标包括焦虑抑郁量表（GAD-7、PHQ-9）
及生理指标（心率变异性、皮肤电导）。所有数据均
采用SPSS 26.0进行统计分析。

为确保实验质量，所有干预人员均接受标准化培训并
通过能力考核。数据收集采用双人录入并交叉核验。
```

### After（白底结构化信息块）

视觉特征：
- 白色背景
- 顶部小标签 "METHODS"，`letter-spacing` 大，灰色
- 左侧竖排 3 个信息块，每块：大数字/关键词 + 右侧 2-3 行说明
- 每个块之间有细线分隔
- 底部注释：详细方法见演讲者备注
- 信息量与 Before 相当，但结构化呈现

内容：
```
METHODS

72 名受试者
随机对照双盲 · 实验组 n=36 vs 对照组 n=36
纳入标准筛选，SPSS 26.0 统计分析
──────────────────────────────
8 周干预方案
每周 2 次 × 45 min · 认知行为训练 + 生物反馈
对照组接受安慰剂控制处理
──────────────────────────────
4 项核心指标
MMSE · MoCA · GAD-7 · PHQ-9
辅以心率变异性、皮肤电导等生理指标

详细方法与质控流程见演讲者备注
```

### Takeaway
1. 段落 → 信息块：每块由一个关键数字引导，后面跟 1-2 行说明
2. 信息量不减少，但从"读论文"变成"扫重点"——3 秒就能抓住要点
3. 统一字色（最多 2 种：正文黑/灰 + 数字强调色），删掉彩色和随机格式

## Case 4：数据表格页

### Before（手残丑）

视觉特征：
- 白色背景
- 标题 "Table 2. Experimental Results"（居中，无结论）
- Excel 直接粘贴风格：粗黑边框
- 表头亮蓝背景 + 白字
- 灰色交替行
- 5 列 × 6 行数据，单位和统计值混在一起
- 底部密排注释

内容：
```
        Table 2. Experimental Results

┌────────┬────────┬────────┬────────┬────────┐
│ Group  │ Pre    │ Post   │ Change │ p-value│（蓝色背景白字）
├────────┼────────┼────────┼────────┼────────┤
│ Exp-A  │ 45.2   │ 68.7   │ +23.5  │ <0.001 │
│ Exp-B  │ 44.8   │ 61.3   │ +16.5  │ 0.003  │（灰色行）
│ Ctrl-1 │ 45.5   │ 47.2   │ +1.7   │ 0.42   │
│ Ctrl-2 │ 44.1   │ 45.8   │ +1.7   │ 0.38   │（灰色行）
│ Sham   │ 45.0   │ 46.1   │ +1.1   │ 0.55   │
└────────┴────────┴────────┴────────┴────────┘
Note: Values are means (SD). Pre=baseline,
Post=8-week. Change=Post-Pre. p-values from
independent t-tests. *p<0.05, **p<0.01
```

### After（白底简化高亮表格）

视觉特征：
- 白色背景
- 标题即结论："实验组 A 效果最显著，提升 52%"（粗体 + 关键数字暖橙色）
- 副标题：前后对比得分，干预 8 周后
- 简化表格：去粗边框，只用细横线分隔
- 关键行（Exp-A）用淡暖橙背景高亮
- 无关行（Sham）用浅灰字弱化
- 底部简短注释

内容：
```
实验组 A 效果最显著，提升 52%     （标题即结论）
前后对比得分（均值），干预 8 周后   （灰色副标题）

  组别      前测    后测    变化     p 值
────────────────────────────────────────
  实验 A    45.2    68.7   +23.5   <0.001  ← 高亮行
  实验 B    44.8    61.3   +16.5    0.003
  对照 1    45.5    47.2    +1.7    0.42    ← 灰字
  对照 2    44.1    45.8    +1.7    0.38    ← 灰字

均值（标准差）· 独立样本 t 检验 · n=36/组
```

### Takeaway
1. 标题写结论而不是编号——"A 组提升 52%" 而不是 "Table 2"
2. 高亮关键行，弱化参考行——让眼睛自动看到最重要的数据
3. 去掉彩色表头和粗边框，用留白和细线分隔——信息不少但不吵

## Case 5：图表页

### Before（手残丑）

视觉特征：
- 白色背景
- 标题只写 "Fig.3 Results"（无结论）
- Excel 默认配色 6+ 种颜色的柱状图
- 带 3D 透视效果
- 密集网格线
- 右侧大面积图例（8 个条目）
- Y 轴标签竖排
- 底部密排注释

内容：
```
        Fig.3 Results of Experiment

  ┌──────────────────────────┐ ■ Ctrl-W1
  │ ██ ██                    │ ■ Exp-W1
  │ ██ ██ ██ ██              │ ■ Ctrl-W4
  │ ██ ██ ██ ██ ██ ██        │ ■ Exp-W4
  │ ██ ██ ██ ██ ██ ██ ██ ██  │ ■ Ctrl-W8
  │ (3D bars, 8 colors)      │ ■ Exp-W8
  └──────────────────────────┘ ■ Sham-W4
   T1 T2 T3 T4 T5 T6 T7 T8   ■ Sham-W8

Note: AU = arbitrary units; error bars = SEM;
n=12/group; *p<0.05
```

### After（白底简洁分组图表）

视觉特征：
- 白色背景
- 标题即结论："处理组在第 8 周达到峰值"
- 分组柱状图只用 2 色（灰 + 暖橙）
- 3 组柱子（W1/W4/W8），每组 2 根
- 关键数据点（W8 实验组）有峰值标注
- 图例内嵌在图表旁边（不单独占位）
- 无 3D、无密集网格
- 底部简短统计注释

内容：
```
处理组在第 8 周达到峰值            （标题即结论）
平均分值；误差棒=SEM；n=12/组       （灰色副标题）

     W1        W4        W8 ★
   ┌──┐┌──┐  ┌──┐┌──┐  ┌──┐┌════┐ ← 峰值标注
   │  ││  │  │  ││  │  │  ││    │
   │灰││橙│  │灰││橙│  │灰││ 橙 │
   └──┘└──┘  └──┘└──┘  └──┘└════┘

   ■ 灰 对照组    ■ 橙 实验组    （图例内嵌）

p<0.001（W8 组间比较）；详细数据见附录
```

### Takeaway
1. 颜色最多 2 种——主数据用强调色，对照用灰色
2. 图例标在数据旁——不要让眼睛在图和图例之间来回跳
3. 标题即结论——观众读标题就知道你的发现是什么

## Case 6：结论页

### Before（手残丑）

视觉特征：
- 白色背景
- 标题蓝色粗体 "Summary & Conclusions"，居中，下方蓝色双线
- 6 条 bullet points 密排
- 每条 1-2 行完整句子
- 底部 "Thank You! 🎉✨🎊" 花体大字 + text-shadow

内容：
```
        Summary & Conclusions
        ═══════════════════════

• 本研究成功建立了一套完整的认知干预评估体系，
  涵盖认知、情感及生理多个维度
• 实验组在干预后各主要指标均显著优于对照组，
  差异具有统计学意义（p<0.001）
• 效应量分析表明干预效果显著（Cohen's d=0.82），
  具有较高的临床意义
• 双盲随机对照设计有效控制了安慰剂效应和
  观察者偏倚，提高了研究内部效度
• 本研究存在一定局限性，包括样本量相对有限、
  随访时间较短及单一地区样本等
• 未来可考虑扩大样本量、延长随访期并在
  多中心开展验证研究

       Thank You! 🎉✨🎊
```

### After（深色分层结论）

视觉特征：
- 深灰 `#1d1d1f` 背景
- 上方小标签 "TAKE-HOME MESSAGES"
- 3 条核心结论：左侧大数字编号（暖橙）+ 右侧主句（白色粗体）+ 下方佐证数据（灰色小字）
- 细线分隔
- "未来方向" 一行（3 个关键词用 · 分隔）
- "局限性" 一行（简短）
- 最底部：邮箱 + 数据集链接（行动号召）
- 无 emoji

内容（信息量与 Before 相当，重新分层）：
```
TAKE-HOME MESSAGES

1  认知干预体系有效
   多维评估（认知+情感+生理）· d=0.82 · p<0.001

2  效果在第 8 周达峰
   8 周干预后实验组显著优于对照 · 需要足够干预时长

3  双盲 RCT 设计可靠
   有效控制安慰剂效应和观察者偏倚

───────────────────────────
未来方向  扩大样本量 · 多中心验证 · 延长随访
局限性    样本量有限（n=72）· 单一地区 · 随访 8 周

zhangsan@university.edu · github.com/dataset
```

### Takeaway
1. 6 条砍成 3 条——工作记忆上限 3-4 条，超过就全忘了
2. 每条分两层：主结论（粗体）+ 佐证数据（小字灰色），信息不丢但有主次
3. 结尾是行动号召（邮箱/链接），不是 Thank You 和 emoji

## 实现注意事项

1. 所有幻灯片内容是 JS 字符串常量（`BEFORE_1` ~ `AFTER_6`），用 inline style
2. Before 幻灯片要"真实的丑"——文字密度要够、颜色要混乱、格式要不统一
3. After 幻灯片要"优雅的满"——信息量相近但结构化清晰、视觉层次分明
4. `.mko-slide` 容器是 16:9 + 28px 32px padding，内容要在这个空间里合理布局
5. Before 白底为主，After 封面/结论深色、中间白底
6. 页面其他部分（Hero、S1 原则、Footer、评分系统）不变，只替换 6 对 BEFORE/AFTER 常量和对应的 takeaway 文案
