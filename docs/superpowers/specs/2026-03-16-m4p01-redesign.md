# M4/P01 设计速成指南 — 重做设计

## 设计目标

将 P01 从"7 个原则平铺罗列"重组为"三层递进"结构：看懂差距 → 学原则 → 动手改。解决内容太浅、缺少"为什么"、交互疲劳的问题。

## 核心原则

**三层递进，符合学习心理学：**
1. 先被差距震撼（情感驱动）
2. 再学为什么（认知驱动）
3. 最后动手改（行为驱动）

## 新页面结构

```
Hero（保持不变）
├─ S1 看懂差距（滚动驱动 Before→After 轮播，4 组）
├─ S2 对比（重交互：5 档滑块 + "为什么"文案）
├─ S3 配色（重交互：方案选择器 + 色板复制 + "为什么"文案）
├─ S4 对齐（轻交互：一键对齐动画演示 + 文案）
├─ S5 留白（轻交互：3 档切换 + 文案）
├─ S6 实战改造（粘性滚动 5 步，串联全部原则）
└─ Footer
```

从 7 个 section 变为 6 个（Hero 不算），删掉了"字体"和"视觉层次"两个独立 section（字体融入配色，视觉层次融入实战改造）。

## S1：看懂差距

### 交互方式

GSAP ScrollTrigger pin 住一个幻灯片展示区域。用户滚动时依次展示 4 组 Before→After：

1. 封面页 Before → crossfade → After
2. 文字页 Before → crossfade → After
3. 图表页 Before → crossfade → After
4. 结论页 Before → crossfade → After

每组之间有切换过渡（scale 0.95 → 1.0 + opacity fade）。

### 旁白文案（每组一句）

| 组 | 文案 |
|----|------|
| 1 | 同样的信息，不同的表达 |
| 2 | 200 字变 3 个关键词——信息量没少 |
| 3 | 8 种颜色变 2 种——数据更清晰 |
| 4 | 6 条结论变 3 条——记住的更多 |

### 幻灯片内容

**复用 P03 的 BEFORE/AFTER 常量**（封面/文字/图表/结论），不重新设计。通过 `import` 或直接复制 HTML 字符串。

### 视觉设计

- 深色背景 section
- 居中展示区域，max-width 720px
- 幻灯片下方显示当前组的文案（fade in/out）
- 左侧或底部显示进度指示器（4 个小圆点，当前高亮）
- Pin 距离约 4 × 100vh（每组 1 屏的滚动量）

## S4：对齐（轻交互）

### 替换当前拖拽画布

当前的拖拽积木块太抽象。替换为：

**一张真实幻灯片的对齐演示：**
- 幻灯片内容：标题 + 3 行要点 + 一个迷你图表 + 一行来源注释
- 初始状态：所有元素位置随机偏移（标题居中、要点缩进不一、图表偏右、注释偏左）
- 两个按钮：
  - "对齐"：点击后所有元素 GSAP 动画滑向左对齐线，辅助线闪现
  - "打乱"：恢复到乱的状态

### 文案

> 看不见的线条是秩序感的来源。所有元素沿同一条隐形线排列，观众的视线就能自然流动，而不是在页面上乱跳。
> 规则很简单：一张 slide 上最多用两种对齐方式（通常是左对齐 + 居中标题）。

### 视觉调整

- 浅色背景 section
- section 标题改为"对齐：看不见的秩序"
- 副标题改为"所有元素沿同一条隐形线排列，视线就能自然流动"

## S5：留白（轻交互）

### 简化当前 5 档为 3 档

- 档 1（过载）：padding 小、字号小、间距紧、有背景花纹
- 档 2（适中）：padding 中等、字号适中、间距合理
- 档 3（极简）：padding 大、字号大、间距充足、只保留核心

3 个圆形按钮切换（1/2/3），当前档高亮。

### 文案

> 留白不是浪费空间，是给内容呼吸的房间。当你删掉一个元素后发现什么都没少——说明它本来就不该在那里。
> 科研 PPT 最常见的错误就是"塞满"。每张 slide 的留白面积应该至少占 30%。

### 视觉调整

- 深色背景 section
- section 标题改为"留白：给内容呼吸的空间"
- 副标题改为"删掉一个元素后什么都没少，说明它本来就不该在"
- 演示幻灯片内容保持不变（数据分析结果）

## S6：实战改造（粘性滚动）

### 交互方式

复用当前 S6 的粘性滚动架构：
- 桌面端：左侧幻灯片固定（居中），右侧步骤文字滚动
- 移动端：顶部 tab 切换
- `.df-snr-wrap` 的 `align-items: flex-start` 保持

### 5 个改造步骤

| Step | 原则 | 标题 | 幻灯片变化 | 解释 |
|------|------|------|-----------|------|
| 0 | — | 原始版 | 全部"噪音"可见：花边框、渐变背景、3 个剪贴画 SVG、Comic Sans 标题带阴影、8 条 bullet（含 5 条冗余）、密集统计表格、花哨页脚 | 这是一张典型的"什么都想放"的幻灯片。信息密度高，但没有任何重点。 |
| 1 | 信噪比 | 删掉噪音 | 边框淡出、渐变淡出、剪贴画淡出 | 不传达信息的装饰就是噪音。边框和剪贴画看起来热闹，但没有一个元素帮助理解内容。 |
| 2 | 对比 | 建立层次 | 标题变大变粗（18→26px，fontWeight 600）、正文缩小、关键数字用强调色突出 | 标题是观众第一眼看到的东西。当标题和正文一样大，观众不知道从哪里开始读。 |
| 3 | 留白 | 精简内容 | 8 条 bullet 变 3 条（冗余项淡出、maxHeight→0）、表格变为简洁柱状图（table 淡出、chart 淡入）、间距增大 | 8 条 bullet 没人看得完。砍到 3 条，每条都是核心发现。剩下的放演讲者备注。 |
| 4 | 配色+对齐 | 统一风格 | 字体从 Comic Sans 变正规字体、颜色统一为主色+强调色、元素左对齐、页脚淡出 | 最后一步把所有细节对齐：字体统一、颜色收敛、元素沿同一条线排列。 |

### 步骤文案模板

每步右侧文字包含：
```
<h3>Step N：标题</h3>
<p>一段解释（2-3 句话，说明改了什么、为什么这样改）</p>
```

- 副标题改为"用刚学的 4 个原则，把一张'什么都想放'的 slide 改到专业级"

## 实现注意事项

1. **GSAP 必须从 ScrollAnimations.js 导入**
2. **S1 滚动驱动**：用 `ScrollTrigger.create({ trigger, pin, scrub })` 实现
3. **S6 粘性滚动**：保持当前 JS transform 实现 + `align-items: flex-start`
4. **所有 scroll listener**：`{ passive: true }`
5. **destroy()** 必须清理所有 ScrollTrigger 和 event handler
6. **移动端**：S1 改为手动切换（4 个 tab），S6 保持 tab 切换
7. **Hero 元素必须有 `opacity:0` 内联样式**
8. **Footer 元素不要有 `opacity:0`**
