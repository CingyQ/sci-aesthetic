### m2-p1 AI 与科研绘图：新范式
**核心叙事**：AI 改变了什么？能做什么、不能做什么？什么时候该用、不该用？
- **S1 范式转变**：传统绘图 vs AI 辅助绘图的工作流对比——不是"AI 更快"，而是思维方式变化（从"画"到"描述+迭代"）

- **S2 能力边界**：AI 擅长（概念示意/风格迁移/快速草图）vs 不擅长（精确数据图/定量标注/复杂机制准确性），用环境科学案例说明
  - 交互式场景判断器：6-8 个科研绘图场景，用户点击判断"该用 AI / 不该用 AI / AI 辅助"，即时反馈正确答案和理由
- **S3 工具生态速览（2026）**：按能力分类而非逐个介绍。三层：①通用生成（GPT Image 1.5 / Midjourney v7 / FLUX 1.1 Pro）②矢量专精（Recraft V3 SVG / StarVector）③结构化图解（Mermaid AI / tldraw / Excalidraw AI）
  - 交互式能力矩阵：横轴=任务类型（概念图/流程图/数据图/图标），纵轴=工具，单元格色块表示能力强弱，hover/点击显示详情
- **S4 环境科学 AI 绘图展示**：用 Nano Banana 生成的真实案例展示
  - **场景切换展示组件**：5 个按钮切换场景（水污染处理工艺 / 碳循环机制 / 遥感分析 pipeline / 微塑料迁移路径 / 生态系统服务评估），每个展示 AI 生成图 + prompt + 简评

### m2-p2 从想法到图像：Prompt 的艺术
**核心叙事**：科研图从脑中概念到 AI 出图的完整过程，重点在结构化 prompt 和迭代优化。
- **S1 Prompt 解剖（CDTF 四要素）**：Context（出版目标/领域）+ Description（精确科学术语）+ Technique（风格/构图/色彩）+ Format（分辨率/比例/色彩空间）。用碳循环示意图 prompt 拆解
  - 粘性滚动步骤流：左侧固定完整 prompt，右侧滚动高亮当前讲解部分，同步展示对生成结果的影响
- **S2 好/差 Prompt 对比**：3 组环境科学场景（①流域水质监测流程图 ②土壤重金属修复机制 ③大气颗粒物来源解析）
  - BeforeAfter 滑块（左差右好）+ 下方 prompt 逐行对比（差异高亮）
- **S3 迭代优化工作流**："湿地生态系统服务评估框架图"从 v1→v2→v3→v4 的迭代，每轮修改一个维度（构图→色彩→细节→标注）
  - 横向时间线：4 版本缩略图 + 点击展开大图 + prompt diff（类似 git diff）+ 修改理由
- **S4 Prompt 模板库**：按科研图类型分类（概念示意图/实验流程图/Graphical Abstract/综述框架图/数据可视化辅助），填空式结构 + 环境科学示例
  - 列表+预览浏览器布局：左模板列表 + 右模板结构+示例 + CopyButton。可替换部分用 `[占位符]` 标注
- 🖼️ **预生成素材**：3 组好差对比各 2 张 + 1 组 4 步迭代序列，共 ~10 张（全部附完整 prompt 文本）

### m2-p3 从位图到矢量：AI 输出的后处理
**核心叙事**：AI 生成的图只是半成品，这一页讲"下半场"——把 AI 粗稿处理成出版级成果。
- **S1 为什么需要后处理**：AI 直接输出的 5 个常见问题（分辨率不足/文字乱码/科学细节错误/风格不统一/格式不合规），每个用环境科学案例图示
  - 滚动驱动 5 张问题卡片：AI 原图 + 红圈标注问题区域 + 文字说明
- **S2 矢量化三条路径（2026）**：①AI 原生 SVG（Recraft V3 直接输出矢量）②AI 辅助转换（StarVector / Vectorizer.AI 位图转 SVG）③传统 Image Trace（Illustrator / Inkscape 参数调节）
  - Tab 切换器：每条路径的流程图 + 适用场景 + 优缺点 + 示例对比
- **S3 Image Trace 参数模拟器**：传统矢量化仍是最常用方案

- **S4 精修工作流**：矢量化后 5 步精修（清理多余路径→统一配色→修正标注→调整布局→导出出版格式），以微塑料迁移路径图为案例

### m2-p4 AI 辅助科研图解设计
